# Persona Null 処理圧力 / Audit Pressure Spec

## 1. 目的

調査を網羅してから裁定することだけが常に最適にならないよう、監査官の行動に応じて都市警備局・都市OS・行政処理期限からの圧力が上昇するターン制指標を追加する。

## 2. 基本原則

- 処理圧力はリアルタイム制限ではなく、プレイヤーの行動によってのみ上昇する。
- 圧力によるゲームオーバー、自動処理、裁定禁止は発生しない。
- 圧力は裁定の正解・不正解を決定しない。
- 圧力は、どのような行政圧力下で判断したかを示す監査記録である。
- 最終値と状態は結果画面および保存結果に記録する。

## 3. 数値ルール

| 行動 | 上昇量 |
| --- | ---: |
| 記憶ノードの初回確認 | +4 |
| 矛盾タグ登録 | +3 |
| 解析アクション実行 | +8 |
| JUDGMENT READY 初回到達 | +5 |

値は 0 から開始し、100 を上限とする。レベルは 0〜29 を LOW、30〜59 を MEDIUM、60〜79 を HIGH、80〜100 を CRITICAL とする。再読、タグ解除、実行済み解析の再操作、JUDGMENT READY 再到達では加算しない。

## 4. 表示と裁定

- 調査画面には現在値、最大値、状態、レベル別説明を表示する。
- HIGH では JUDGMENT CONSOLE に遅延警告を表示する。
- CRITICAL では行政処理への移送警告を赤系表示するが、実際の強制処理は行わない。
- 裁定画面には「処理圧力下での裁定」と現在値を表示する。
- 結果画面には最終値、状態、直近5件の圧力イベントを表示する。

## 5. 保存互換性

`SavedCaseResult.auditPressure` は任意項目とする。新規裁定では最終値とレベルを保存するが、同項目を持たない既存 localStorage 結果も有効な保存結果として読み込む。
